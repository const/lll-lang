This file contains the current plan for the project.

  1. Implement ETL language for LLVM assembler (will be use for asm sections). Write compiler to from ETL LLVM ASM to LLVM ASM that runs on JVM (Scala?).

    1. Simple language core with low level types
    2. Structs
    3. Tuples
    4. Unions
    5. Arithmentics
    6. Jumps/conditions

  2. Raw (Non-GC) library

    1. Unit Testing and FFI
    2. Binary IO
    3. Encoding Support (Unicode, Ascii, One-byte) (FFI) and IO
    4. Non-GC collections
    5. Attempt non-GC implementation of ETL Parser
  3. Look at OpenJDK GC
  4. Try to write a similar GC in BitC-like LLL language, designing language during the process
  5. Try to write compiler from LLL to LLVM ASM

  6. Try to write ETL parser using language extensions for GC objects

    1. GC Streams, GC Closures, GC Collections, Lazy
    2. Resolver infrastructure
    3. ETL parser
    4. Min XML parser
    5. XML catalog support
  7. Try to write self-hosting compiler to bytecode
